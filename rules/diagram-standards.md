# Diagram Standards

> *This specification uses terms defined in [RFC 2119](https://datatracker.ietf.org/doc/html/rfc2119).*
> *This rule is enforced during `/report` and `/exposition` commands.*

---

## 1. Mandatory Tool

- **DR-01:** All diagrams **MUST** be created using [Mermaid](https://mermaid.js.org/) syntax.
- **DR-02:** LaTeX-native diagram environments (`tikzpicture`, `pgfplots`, `circuitikz`, `pstricks`) **MUST NOT** be used for diagram generation. LaTeX is for document structure and typesetting only.
- **DR-03:** The Mermaid CLI (`mmdc`) **MUST** be used to convert `.mmd` files to SVG for inclusion.
- **DR-04:** The `--background` / `-b` flag **MUST** be set to `transparent` in all `mmdc` invocations to prevent white-background clipping in different document viewers.
- **DR-05:** Each diagram **MUST** be defined in its own `.mmd` file inside the `diagrams/` directory.

---

## 2. Workflow

1. **Define** the diagram in a `.mmd` file under `diagrams/` using Mermaid syntax.
2. **Render** to SVG:
   ```bash
   mmdc -i diagrams/diagram.mmd -o diagrams/diagram.svg -b transparent
   ```
3. **Include** in LaTeX:
   ```latex
   \usepackage{svg}
   ...
   \includesvg[width=0.9\linewidth]{diagrams/diagram}
   ```
4. **Reference** in Markdown reports:
   ```markdown
   ![Diagram description](diagrams/diagram.svg)
   ```

---

## 3. Supported Diagram Types

| Type | Mermaid Syntax | Use Case |
|------|---------------|----------|
| Flowchart | `graph TD` / `graph LR` | Algorithms, workflows, processes |
| Sequence diagram | `sequenceDiagram` | Interactions between components |
| Class diagram | `classDiagram` | Data models, OOP structures |
| State diagram | `stateDiagram-v2` | State machines, FSM |
| Entity-relationship | `erDiagram` | Database schemas |
| Block diagram | `block-beta` (Mermaid v11+) | System architecture, hardware blocks |
| Pie chart | `pie` | Data distribution, statistics |
| Gantt | `gantt` | Project timelines, planning |

---

## 4. File Organization

```
diagrams/
├── diagram_01.mmd       # Mermaid source
├── diagram_01.svg        # Rendered output (generated by mmdc)
├── diagram_02.mmd
└── diagram_02.svg
```

- Source `.mmd` files **MUST** be committed to version control.
- Rendered `.svg` files **SHOULD** be committed (so the document compiles without `mmdc`).
- The `diagrams/` directory is created at the project root when the command is invoked.

---

## 5. Quality Checks

Before marking a task as complete, the agent **MUST** verify:

- [ ] All `.mmd` files render without errors: `mmdc -i <file> -o test.svg -b transparent`
- [ ] No `\begin{tikzpicture}` appears in any `.tex` file.
- [ ] Every figure reference in the document corresponds to a generated SVG file.
- [ ] Rendered SVGs contain transparent backgrounds (verify with `head` or file properties).
- [ ] No placeholder diagram uses `\includegraphics[width=...]{diagrams/...}` without the file existing.
